Gordon

Gordon is affectionately described as "A C kitchen sink"


Basically, this library started as a learning exercise. I'm aiming to implement the full C99 Standard library. This includes all needed headers and assembly chunks for the library to work on x86-64 and 64-bit ARM.

In addition, the gord.h header will provide a number of general purpose data structures for alternative uses.

My planned features include:

  • C99 compliant Standard Library, closely coupled to the Linux kernel
  • pthreads and unistd
  • Simple inline assembly use, when it makes sense

This is my most active project so feel free to follow my progress along!

Spork

Over the last year I've been experimenting with the Rust systems language. To augment some of my projects, I developed Spork as a key-value store.


Essentially, my goal for this library is to reimplement Redis but as an internal component of the library.

This will include features such as timed-serialization in the near future

Dunwich

A while ago, I needed to stress-test a series of wifi adapaters against (possibly flaky) Linux drivers.

Dunwich is a small Python utility to spawn a server on the machine with the adapter being tested. Then, the script can be run as a client on other machines on the network. These client-server connections are hammered by transmitting the full text of The Dunwich Horror by H.P. Lovecraft back and forth in randomized order.

This utility proved useful in ruling out adapters that would crash under moderate load

Py-Bidchips

I've had an interest in Bioinformatics for a number of years as a research area. For my Honour's Project, I worked with the Perkins lab at the Ottawa Hostpital Research Institute.

I ended up developing a Python implementation of the BIDCHIPS application for GPU-Accelerated statistical analysis of ChIP-Seq data. BIDCHIPS aims to remove inherrent statistical bias from ChIP-Seq data through regression models.

I implmented the GPU acceleration using the ArrayFire library. I was able to achieve 2-3x speed improvements under certain workloads relative to the pure CPU version.


This project isn't open-source yet but I hope to have it soon....
